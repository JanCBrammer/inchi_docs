81 * Notes on using INCHI_BOND_STEREO_SINGLE_* from inchi_BondStereo2D *
82 * *
83 * These stereo markings are used by InChI to characterize a stereogenic *
84 * atom if and only if all neighbors of this atom have same z-coordinate *
85 * as this atom (that is, in case of 2D fragment). *
86 * The only exception is INCHI_BOND_STEREO_SINGLE_?EITHER marking which *
87 * always assigns to the atom an "unknown" parity (u). *
88 * *
89 * Note the behavior which is default for InChI software v.1.04/03/02std *
90 * (at -NEWPSOFF option is not supplied) 2D stereo interpretation: *
91 * only bonds that have sharp end pointing to the stereogenic atom are *
92 * considered as being out of plane and only sharp ends of *
93 * INCHI_BOND_STEREO_SINGLE_?EITHER bonds are considered to determine *

157/*******************************************************************
158 * Notes: 1. Atom ordering numbers (i, k, and atom[i].neighbor[j] below)
159 * start from zero; max. ordering number is (num_atoms-1).
160 * 2. inchi_Atom atom[i] is connected to the atom[atom[i].neighbor[j]]
161 * by a bond that has type atom[i].bond_type[j] and 2D stereo type
162 * atom[i].bond_stereo[j] (in case of no stereo
163 * atom[i].bond_stereo[j] = INCHI_BOND_STEREO_NONE)
164 * Index j is in the range 0 <= j <= (atom[i].num_bonds-1)
165 * 3. Any connection (represented by atom[i].neighbor[j],
166 * atom[i].bond_type[j], and atom[i].bond_stereo[j])
167 * should be present in one or both adjacency list:
168 * if k = atom[i].neighbor[j] then i may or may not be present in
169 * atom[k].neighbor[] list. For example, the adjacency lists may be
170 * populated with only such neighbors that atom[i].neighbor[j] < i
171 * All elements of an adjacency list must be different, that is,
172 * a bond must be specified in an adjacency list only once.
173 * 4. in Molfiles usually
174 * (number of implicit H) = Valence - SUM(bond_type[])
175 * 5. Seemingly illogical order of the inchi_Atom members was
176 * chosen in an attempt to avoid alignment problems when
177 * accessing inchi_Atom from unrelated to C programming
178 * languages such as Visual Basic.

182 0D Stereo Parity and Type definitions
183 *******************************************************************
184 Note:
185 =====
186 o Below #A is the ordering number of atom A, starting from 0
187 o See parity values corresponding to 'o', 'e', and 'u' in
188 inchi_StereoParity0D definition below)
189
190 =============================================
191 stereogenic bond >A=B< or cumulene >A=C=C=B<
192 =============================================
193
194 neighbor[4] : {#X,#A,#B,#Y} in this order
195 X central_atom : NO_ATOM
196 \ X Y type : INCHI_StereoType_DoubleBond
197 A==B \ /
198 \ A==B
199 Y
200
201 parity= 'e' parity= 'o' unknown parity = 'u'
202
203 Limitations:
204 ============
205 o Atoms A and B in cumulenes MUST be connected by a chain of double bonds;
206 atoms A and B in a stereogenic 'double bond' may be connected by a double,
207 single, or alternating bond.
208 o One atom may belong to up to 3 stereogenic bonds (i.g. in a fused
209 aromatic structure).
210 o Multiple stereogenic bonds incident to any given atom should
211 either all except possibly one have (possibly different) defined
212 parities ('o' or 'e') or should all have an unknown parity 'u'.
213
214 Note on parities of alternating stereobonds
215 ===========================================
216 D--E
217 In large rings (see Fig. 1, all // \\
218 atoms are C) all alternating bonds B--C F--G
219 are treated as stereogenic. // \\
220 To avoid "undefined" bond parities A H
221 for bonds BC, DE, FG, HI, JK, LM, AN \ /
222 it is recommended to mark them with N==M J==I
223 parities. \ /
224 L==K Fig. 1
225 Such a marking will make
226 the stereochemical layer unambiguous
227 and it will be different from the B--C F--G
228 stereochemical layer of the second // \\ // \\
229 structure (Fig. 2). A D--E H
230 \ /
231 N==M J==I
232 By default, double and alternating \ /
233 bonds in 8-member and greater rings L==K Fig. 2
234 are treated by InChI as stereogenic.

238 tetrahedral atom
239 =============================================
240
241 4 neighbors
242
243 X neighbor[4] : {#W, #X, #Y, #Z}
244 | central_atom: #A
245 W--A--Y type : INCHI_StereoType_Tetrahedral
246 |
247 Z
248 parity: if (X,Y,Z) are clockwize when seen from W then parity is 'e' otherwise 'o'
249 Example (see AXYZW above): if W is above the plane XYZ then parity = 'e'

251 3 neighbors
252
253 Y Y neighbor[4] : {#A, #X, #Y, #Z}
254 / / central_atom: #A
255 X--A (e.g. O=S ) type : INCHI_StereoType_Tetrahedral
256 \ \
257 Z Z
258
259 parity: if (X,Y,Z) are clockwize when seen from A then parity is 'e',
260 otherwise 'o'
261 unknown parity = 'u'
262 Example (see AXYZ above): if A is above the plane XYZ then parity = 'e'
263 This approach may be used also in case of an implicit H attached to A.

265 =============================================
266 allene
267 =============================================
268
269 X Y neighbor[4] : {#X,#A,#B,#Y}
270 \ / central_atom : #C
271 A=C=B type : INCHI_StereoType_Allene
272
273 Y X
274 | |
275 when seen from A along A=C=B: X-A Y-A
276
277 parity: 'e' 'o'
278
279 parity: if A, B, Y are clockwise when seen from X then parity is 'e',
280 otherwise 'o'
281 unknown parity = 'u'
282 Example (see XACBY above): if X on the diagram is above the plane ABY
283 then parity is 'o'

285 Limitations
286 ===========
287 o Atoms A and B in allenes MUST be connected by a chain of double bonds;

289
290 How InChI uses 0D parities
291 ==========================
292
293 1. 0D parities are used if all atom coordinates are zeroes.
294
295 In addition to that:
296
297 2. 0D parities are used for Stereobonds, Allenes, or Cumulenes if:
298
299 2a. A bond to the end-atom is shorter than MIN_BOND_LEN=0.000001
300 2b. A ratio of two bond lengths to the end-atom is smaller than MIN_SINE=0.03
301 2c. In case of a linear fragment X-A=B end-atom A is treated as satisfying 2a-b
302
303 0D parities are used if 2a or 2b or 2c applies to one or both end-atoms.
304
305 3. 0D parities are used for Tetrahedral Atoms if at least one of 3a-c is true:
306
307 3a. One of bonds to the central atom is shorter than MIN_BOND_LEN=0.000001
308 3b. A ratio of two bond lengths to the central atom is smaller than MIN_SINE=0.03
309 3c. The four neighbors are almost in one plane or the central atom and
310 its only 3 explicit neighbors are almost in one plane
311
312 Notes on 0D parities and 'undefined' stereogenic elements
313 =========================================================
314
315 If 0D parity is to be used according to 1-3 but CH3 CH3
316 has not been provided then the corresponding \ /
317 stereogenic element is considered 'undefined'. C=CH
318 /
319 For example, if in the structure (Fig. 3) H
320 the explicit H has been moved so that it Fig. 3
321 has same coordinates as atom >C= (that is,
322 the length of the bond H-C became zero)
323 then the double bond is assigned 'undefined' CH3 CH3
324 parity which by default is omitted from the \ /
325 Identifier. CH=CH
326
327 However, the structure on Fig. 4 will have double Fig. 4
328 bond parity 'o' and its parity in the Identifier is (-).
329
330 Notes on 0D parities in structures containing metals
331 ====================================================
332 Since InChI disconnects bonds to metals the 0D parities upon the
333 disconnection may change in several different ways:
334
335 1) previously non-stereogenic bond may become stereogenic:
336
337 \ / \ /
338 CH==CH disconnection CH==CH
339 \ / ======>
340 M M
341
342 before the disconnection: after the disconnection:
343 atoms C have valence=5 and the double bond may become
344 the double bond is not stereogenic
345 recognized as stereogenic
346
347 2) previously stereogenic bond may become non-stereogenic:
348
349 M M(+)
350 \ / /
351 N==C disconnection (-)N==C
352 \ ======> \
353
354 3) Oddball structures, usually resulting from projecting 3D
355 structures on the plane, may contain fragment like that
356 depicted on Fig. 5:
357
358 M A M A
359 |\ / B / B
360 | X / disconnection / /
361 |/ \ / ======> / /
362 C===C C===C
363 Fig. 5
364 (X stands for bond intersection)
365
366 A-C=C-B parity is A-C=C-B parity is
367 trans (e) cis (o) or undefined
368 because the bond because C valence = 3,
369 orientation is same not 4.
370 as on Fig, 6 below:
371
372 A M
373 \ / Removal of M from the structure
374 C===C on Fig. 5 changes the geometry from trans
375 / \ to cis.
376 M' B Removal of M and M' from the structure
377 Fig. 6 on Fig. 6 does not change the A-C=C-B
378 geometry: it is trans.
379
380 To resolve the problem InChI API accepts the second parity
381 corresponding to the metal-disconnected structure.
382 To store both bond parities use left shift by 3 bits:
383
384 inchi_Stereo0D::parity = ParityOfConnected | (ParityOfDisconnected<<3)
385
386 In case when only disconnected structure parity exists set
387 ParityOfConnected = INCHI_PARITY_UNDEFINED.
388 This is the only case when INCHI_PARITY_UNDEFINED parity
389 may be fed to the InChI.
390
391 In cases when the bond parity in a disconnected structure exists and
392 differs from the parity in the connected structure the atoms A and B
393 should be non-metals.
394
